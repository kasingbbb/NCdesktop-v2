# NCdesktop 20步开发框架宪章 (The 20-Step Development Framework)

> 适用产品：Omni / Arca 多模态知识采集终端 — 桌面中枢 **NCdesktop**
> 阶段定位：项目立项后的【规范化流水线施工图纸】
> 前置依据：《NCdesktop 软件 PRD》、《Liquid Glass UI 宪章》、《并行开发策略》、《Speckit 项目开发宪章》
> 版本：1.1.0 | 生效日期：2026-03-25

---

## 编者按 (Charter Preamble)

本宪章是将 NCdesktop 宏大的“多模态时间轴”、“全局悬浮窗”、“端侧 AI 与大模型桥接”和“默认离线”安全等各种宏观要求，转化为**不可打破顺序、环环相扣的 20 个物理开发步骤**。

**核心约束：**
1. **严格的 DoD 门禁校验**：凡未满足本宪章当前 Steps (比如 Step 4 接口契约) 的 DoD (Definition of Done)，绝对不允许进入受其依赖的后续 Step (比如 Step 5 或 9)。
2. **三轨并行原则**：从 Step 4 开始，项目正式进入 **Agent α** (Rust核心与数据)、**Agent β** (React交互与动画) 和 **Agent γ** (独立功能与大模型) 的并行模式。汇合点 (如 C3) 产生必须联调。

---

## 📂 阶段一：筑基与基建 (Foundation) | Steps 1-4
**参与者**：全体串行
**核心目标**：从零起步建立一个满足“苹果质感美学”、“离线第一沙盒安全”的工程基底。

### Step 1: `tauri-v2` 骨架设定与 UI 主动注入
- **任务目标**：完成符合端侧 AI 环境且无边框的沉浸式 Tauri v2 初始化。
- **技术要点**：
  - 基于 React 19 + TypeScript + Vite + pnpm。
  - 在 `tauri.conf.json` 中配置 macOS 独有的 `transparent: true` 和 `Overlay` 标题栏。
  - 将《Liquid Glass UI 设计宪章》的所有基础 Token (颜色、毛玻璃滤镜层、阴影尺寸) 注册进入 `Tailwind CSS v4` 的 Theme 中。
- **DoD**：本地可冷启动一个具有纯粹海军蓝 (#1F456E) 背景、支持 macOS 窗口拖拽调整但无原系统标题边框的空白窗口。

### Step 2: 隐私守护神 —— 纯本地 SQLite 构建
- **任务目标**：落实“Default Offline”原则，搭建 SQLite 持久化引擎。
- **技术要点**：
  - 在 Rust 端引入 `rusqlite` 或轻量 ORM。
  - 创建属于该 App 的独立系统应用支持库隔离沙盒 (`~/Library/Application Support/com.notecapt.app/db.sqlite`)。
  - 创建最初的 `Projects` 和 `Assets` 两张基础表并编写数据库 Migration 机制。
- **DoD**：Rust 端的单元测试 (Unit Test) 可成功完成 CRUD，验证沙盒写入正常。

### Step 3: 文件系统沙盒映射机制 (FS Middleware)
- **任务目标**：允许 React 端极速读取动辄几百MB甚至上GB的照片、录音，而不破坏安全机制。
- **技术要点**：
  - 使用 Tauri `fs` 插件建立从安全持久化文件夹到前端 URL 的自定义协议映射 (Custom Protocol 或 `convertFileSrc`)。
  - 禁止前端通过直接传递任意的系统绝对路径读写，全部必须由 Rust 层的受限命令 (IPC) 处理。
- **DoD**：把一张高清图片放在安全数据目录内，React 能够成功将它安全渲染在页面上。

### Step 4: [关键节点 C1] 前后端 IPC 契约对齐
- **任务目标**：这是多 Agent 分头并行的**分水岭**，定义所有的“通用语言”。
- **技术要点**：
  - 编写并确定完整的全局 TypeScript 数据模型 Interface，如 `Timeline`, `AudioTrack`, `Keyframe`, `LiveKnowledgePack` 等模型（详见PRD附录）。
  - 在 Rust 端与 React `lib/` 层编写 IPC Commands Mock，模拟 `invoke('get_timeline')` 将来会返回的数据树。
- **DoD 审查**：α 和 β 两条线的负责开发者/Agent 对齐无误。并建立模拟仓库。

## 🗄️ 阶段二：底层底座双轨并行 (Core Mechanics) | Steps 5-11
**参与者**：开始执行 `Agent α` 和 `Agent β` 双任务线分离。

### [α 线] Step 5: 全家桶业务数据实体持久化
- **任务目标**：Rust 端实现《PRD 3.2》所有的强关系数据操作。
- **技术要点**：完成含外键依赖的 `Tags`, `Notes`, `TimelineSession`, `TrackSegments` 等 SQL 表和复杂的 JOIN 检索逻辑包装。
- **DoD**：Rust 测试用例能向表中一次性填充包含音频时间、多张照片及关系的 Dummy 对象，并零错误地以 JSON 返回。

### [α 线] Step 6: 实体世界感知 —— 硬件热插拔与挂载驱动
- **任务目标**：对 Arca TF 卡接入进行轮询和监听感知。
- **技术要点**：对指定的便携存储卷使用 `fs-watch` 检测，精准找到包含 `.arca/manifest.json` 配置特征的 TF卡目录，并立刻向前端广播特定 Tauri Event。
- **DoD**：插拔物理测试优盘时，控制台能在毫秒级打印出“成功检测知识终端 TF 卡：[序列号]”。

### [α 线] Step 7: TF 卡全量/增量解析与导入引擎
- **任务目标**：读取硬件端已经过 NPU 打标分类的文件树。
- **技术要点**：
  - 建立后台异步 I/O 线程池（Thread Pool）转移巨大媒体文件到本地。
  - 解析 `live_clips` (实况片段)、`photos/*.meta.json` 中保存的关键时间戳及分类状态，把它们消化进 Step 5 的结构库。
- **DoD**：能于 5 秒内将包含 100 张照片及附随 Meta 元数据的测试目录，无阻塞转移同步并录入本地 SQLite。

### [α 线] Step 8: 本地知识库闪电检索服务
- **任务目标**：启用 SQLite 全文检索库。
- **技术要点**：采用 `FTS5` 及高效分词器。创建视图建立项目名、全文大段 OCR 内容、Whisper 离线转写文本的反向索引聚合检索查询 API。
- **DoD**：能够在 1 万条以上的假文本中，200ms 内查询并反馈关联对象的详细信息列表。

### [β 线] Step 9: Liquid Glass 基础与重型组件库构建
- **任务目标**：React UI 视觉组件的具象化。
- **技术要点**：全面实现《Liquid Glass UI 设计宪章》。编码：`GlassCard`、带辉光的 `PrimaryButton`、深色毛玻璃大背景框。
- **DoD**：有一个独立的 Storybook 风格演示路由组件 `#/components` 可审查所有的金色高亮、同心圆效果及毛玻璃呈现。

### [β 线] Step 10: 软件骨架与路由体系搭建
- **任务目标**：根据《PRD 5.3》，完成三大区域（侧边栏、主内容区、Inspector属性面板）布局。
- **技术要点**：
  - 侧边栏及右侧可支持拖拽改变宽度的边界线阻尼调整。
  - 按下项目时的主页面路由刷新。
- **DoD**：左中右的三栏静态布局全部排布成功。

### [β 线] Step 11: Zustand 统一状态树 (Store) 初始化
- **任务目标**：将 Mock 的知识库数据喂给 Step 10 的 UI 骨架。
- **技术要点**：建立 `useLibraryStore`、`useTimelineState Store`。确保所有组件的数据流是单向的，不再各自向外发起重复的数据请求操作。
- **DoD**：组件与 Store 对接成功，无须调用 Rust 层便能在前端演示“点击不同项目，右侧更新元数据”的视觉流程。

## ⚙️ 阶段三：硬核引擎注入 (The Engines) | Steps 12-14
**参与者**：`Agent α`（处理系统音视频解码和极值阵列计算难度较高的工作）

### [α 线] Step 12: Rust 原生流式音频解码与缓冲器
- **任务目标**：解析庞大录音，而不导致前端出现卡死现象。
- **技术要点**：通过 Rust `symphonia` 类库（或直接系统编解码库框架）读取并校验 M4A/WAV 头，给出前端所需的 Sample Rate, Channels 和纯粹的 Duration。
- **DoD**：成功在 Rust 后端以几毫秒时间精确识别一段 1 小时级别录音文件的时间长度与格式信息。

### [α 线] Step 13: Edge AI 实况标签微解码
- **任务目标**：特殊硬件实况知识逻辑的数据整理。
- **技术要点**：提取硬件终端打标的“由图寻音”专有信息——即拍照瞬间向前缓冲 10s 和向后 10s 的“切片相对时间偏移结构”。
- **DoD**：输入一张含有专有 Meta 的照片文件，输出其相对于原 90 分钟长音频起点的真正 Timeline Absolute Offset(ms)。

### [α 线] Step 14: [关键节点 C3] 波形能量向量 (Waveform) 预生成器
- **任务目标**：消除前端解析并渲染音频波形带来的极致性能噩梦（秒开保障）。
- **技术要点**：在硬件内容导入阶段，后台线程自动抽取录音的音量震幅极值（如按 100ms 切分，记录极值产生 Array）。把它以 `.bin` 格式写入磁盘。
- **DoD 审查**：后端能生成仅数 KB 大小的极值二进制文件供前端 `Float32Array` 拉取。此时触发 **C3 架构大合并点**，通知 `Agent β` 接入真实数据。

## ✨ 阶段四：不可思议的灵魂交互 (Magic Experience) | Steps 15-18
**参与者**：`Agent β` 和 独立模块 `Agent γ`。重头戏：时空记忆轴核心模块（F5 特性）。

### [β 线] Step 15: Canvas 时空波形轴一期基础排线
- **任务目标**：完成性能极高的非 DOM 时间线轴呈现。
- **技术要点**：
  - 使用 HTML5 Canvas。基于 `requestAnimationFrame` 绘制背景，利用 S14 的 Float32Array 数组绘制海洋蓝波形背景，以及在已播放的区域填充为亮金色。
  - 支持滚轮放大缩小刻度。
- **DoD**：时间线能够如股票 K 线图一样顺滑左右无限拖动、无掉帧感，且播放头可指示移动。

### [β 线] Step 16: Canvas 智能关键帧节点投射
- **任务目标**：让文本和扫描卡片作为附着点落在对应的时刻之上。
- **技术要点**：将 Step 13 解析出来的绝对相对位置转换成 Canvas 上的 X 坐标，在时间轴上方渲染照片缩略图和卡片，悬停支持缩放放大动态效果。
- **DoD**：长长的背景波形上方，成功吸附呈现出由硬件导入的照片缩略图（小相机金牌 icon）。

### [β 线] Step 17: "The Magic Moment" 视听双向联动引擎
- **任务目标**：产品终极灵魂交互——“随音现图”和“由图寻音”。
- **技术要点**：
  - 交互 1：“由图寻音”。当鼠标点击某张关键帧卡片时，播放头进行 Pre-Roll（后退默认5秒）动画平铺，驱动 Howler.js/AudioContext 将音频在特定 ms 处重新响起，照片同时从卡片升维变大居中。
  - 交互 2：“随音现图”。当录音播放的 currentTime 滑过特定锚点，对应的关键帧自动亮起“Liquid Glass 金色共振光圈”提示动效。
- **DoD**：完成全套动画流程。交互流畅度需接受审查，满足沉浸式回溯回忆的使用场景。

### [γ 线独立] Step 18: F4 全局黑洞悬浮窗 (The Global Dropzone)
- **任务目标**：脱离主窗框架的桌面入口挂件。
- **技术要点**：新建 `Tauri window`，将其注入操作系统的始终置顶显示池，形状圆润半透明，接收外部 File Drop 掉落事件。捕捉到素材后隐退至后台调用分类打标 IPC。
- **DoD**：当向这块屏幕上漂浮着的类“苹果材质常驻垃圾桶”小挂件拖入 PDF 时，它会有形变脉冲反馈效应并触发内部项目导入流程。

## 🚀 阶段五：连接大模型与交付发布 (LLM & Delivery) | Steps 19-20
**参与者**：`Agent γ` 和 全体

### [γ 线] Step 19: F6 模块 —— OpenAI Proxy 穿透大桥
- **任务目标**：安全且结构化地向外界大语言模型递交知识，进行一键提取（不触露本地原音视频）。
- **技术要点**：
  - 构建转换器，将选定范围内的 Timeline + 转写和文字转出为特定的结构化 Markdown。
  - **后端 OpenAI Proxy直连模块**：Rust 层根据配置中的 `OPENAI_API_KEY` 通过 `async-openai` 等请求 Chat/Summarize 端点。
  - 接受远端 SSE 流媒体传输，发出 Tauri Event 回源至给前端，实现流式文字逐字展露的打字机交互面板（AI Analysis 页签）。
- **DoD**：UI 发起一段课堂复习知识获取动作，它能够脱离外部软件依赖，顺畅流式地吐出一篇学习摘要总结并显示在侧边栏 Inspector 侧面板中。

### [全局] Step 20: “零摩擦”封装、全真联调与产物分发
- **任务目标**：所有功能大集结、合规以及产出安装包。
- **技术要点**：
  - 前端体积切割（Rollup）、Rust 端生产编译。确保 `NCdesktop` 内存空闲低于 80MB，冷启动低于 500ms（《PRD》性能指标要求）。
  - 执行安全扫描与沙盒隔离确保策略无漏洞配置。
  - 使用 `tauri build`，绑定产品 Identifier 签发和高清 `icns` 图标封装，目标产出 Apple 环境用的 `dmg` 镜像（涵盖 M 系列处理器的 universal 架构）。
- **DoD**：编译无谬误。在真实的 macOS 实体真机上，通过下载、挂载然后拖动到 Applications 文件内安装并在 Launchpad 中顺利点开、完成体验。
